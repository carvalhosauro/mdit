# mdit MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Editor de markdown TUI em Go com live render inline por bloco, wikilinks com vault, zen mode read-only e fuzzy finder.

**Architecture:** Bubble Tea (Elm architecture) com widget de editor próprio. Documento = linhas cruas; goldmark parseia em `[]Block` cobrindo todas as linhas; bloco sob cursor exibe markdown cru, demais renderizados via lipgloss com cache e viewport virtualizado (prefix sums de alturas). Vault = índice em memória nome→path.

**Tech Stack:** Go ≥1.26, bubbletea v1, lipgloss v1, bubbles, goldmark v1 (+extensão GFM), chroma/v2, go-runewidth, muesli/reflow, charmbracelet/x/exp/teatest.

**Spec:** `docs/superpowers/specs/2026-07-16-mdit-mvp-design.md` — leia antes de qualquer task.

## Global Constraints

- Module path: `github.com/carvalhosauro/mdit`. Binário: `mdit`.
- `CGO_ENABLED=0` sempre (cross-compile limpo). Nenhuma dep com CGO.
- Licença MIT, copyright `2026 Gustavo Carvalho`.
- Todo commit: `gofmt` limpo, `go vet ./...` limpo, `go test ./...` verde.
- Commits em conventional commits (`feat:`, `test:`, `chore:`, `docs:`).
- Cursor SEMPRE em coordenadas cruas `(linha, coluna-em-runes)`; tela é derivada.
- Keybindings do spec: Ctrl+S salvar, Ctrl+Q sair, Ctrl+P fuzzy finder, Ctrl+E zen, Ctrl+] seguir link, Ctrl+B voltar, Ctrl+Z/Ctrl+Y undo/redo.
- Mensagens de UI em inglês (projeto OSS): "unsaved changes", "broken link", etc.
- Cada task termina com deliverable testável + commit. TDD: teste antes da implementação.

### Task 4: `internal/mdparse` — goldmark → blocos + extensão wikilink

**Files:**

**Interfaces:**
- Produces:

```go
type Kind int

const (
	Blank Kind = iota
	Heading
	Paragraph
	CodeFence
	List
	Table
	Blockquote
	ThematicBreak
	Frontmatter
	Other // HTML block etc.
)

type Block struct {
	Kind       Kind
	Start, End int      // range de linhas cruas, inclusivo
	Node       ast.Node // nil para Blank e Frontmatter
}

type Result struct {
	Blocks []Block

}

// Gaps entre blocos do goldmark viram blocos Blank.

// Extensão goldmark: [[target]] e [[target|alias]] viram inline node
type WikiLink struct { ast.BaseInline; Target, Alias string }
func (n *WikiLink) Label() string // Alias se não-vazio, senão Target
var WikiLinkExt goldmark.Extender // registrado com prioridade acima do link parser

// Para follow-link no editor (linha crua sob cursor):
func WikiLinkAt(line string, col int) (target string, ok bool) // col em runes; dentro de [[...]]
```

- [ ] **Step 1:** Testes de `Parse` (table-driven): documento com heading, parágrafo multi-linha, blank, fence com `# fake heading` dentro (deve ser 1 bloco CodeFence), lista aninhada (1 bloco List cobrindo todas as linhas), tabela, blockquote, `---` thematic break, frontmatter YAML no topo (linhas `---`…`---` → 1 bloco Frontmatter). Assert central (helper `assertCoverage(t, res, len(lines))`): blocos contíguos cobrindo tudo. Frontmatter: detectado manualmente ANTES do goldmark (linha 0 == `---`, próxima `---` fecha), parseia só o resto e desloca ranges.
- [ ] **Step 2:** FAIL → implementar `parse.go`:
  - `Source = []byte(strings.Join(lines, "\n"))` + offsets por linha (prefix sum de len+1).
  - Walk dos filhos diretos do root; range de linhas: para nós com `Lines().Len() > 0`, converte offset início/fim → índice de linha; para containers (List, Blockquote) sem Lines, min/max recursivo dos descendentes. ThematicBreak não tem Lines → range = linha do segmento anterior+…: use posição via busca do padrão? Não — solução: ordene blocos conhecidos e preencha gaps; ThematicBreak entre gaps vira bloco de 1 linha detectado por regex `^ {0,3}(-{3,}|\*{3,}|_{3,})\s*$` no gap. Gaps restantes → Blank (linha em branco) ou Other (conteúdo não mapeado).
- [ ] **Step 3:** PASS.
